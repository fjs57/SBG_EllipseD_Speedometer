# Logger Speed

Real-time speed monitoring and data logging application for vehicle field tests, using the **SBG Systems ELLIPS-D** Inertial Navigation System (INS) as the speed and position sensor.

---

## Purpose

During vehicle field tests, the driver must maintain a precise target speed over a route.  
Logger Speed provides:

- a large **speedometer gauge** that auto-scales around the target speed and highlights the acceptable zone in green
- a **real-time speed plot** showing the actual speed against the target over a 60-second rolling window
- **GPS coordinates** displayed with centimetre-level resolution
- **INS solution status** (AHRS, INS Velocity, INS Position, …)
- **CSV data logging** with one file per test run, recording speed, position and the full decoded INS solution status

---

## Features

| Feature | Detail |
|---|---|
| Auto-scaling gauge | Range centres on the nearest integer to the target speed; scale adapts automatically |
| Target zone | Green arc on the gauge and dashed line on the plot mark the ± tolerance band |
| Unit toggle | Switch between **m/s** and **km/h** at any time |
| Signed speed support | Reversing shows as positive magnitude on gauge and plot; signed value is stored in the log |
| Rolling plot | 60-second time window, absolute speed vs. time |
| GPS display | 7 decimal places (≈ 1 cm resolution at the latitude of Brussels) |
| Solution status | Colour-coded INS mode (grey → amber → blue → green) |
| CSV logging | 30 columns: timestamp, speed, position, decoded solution mode + all aiding flags |
| Console log | In-app coloured log panel with levels TRACE / DEBUG / INFO / WARNING / ERROR / FATAL |
| RX indicator | Blinking LED + messages/second counter confirms data is arriving from the INS |
| Multi-monitor safe | Correct rendering on mixed-DPI setups; auto-recalibrates when moving between screens |
| Standalone exe | Single `LoggerSpeed_*.exe` — no installation or admin rights required |

### Key design decisions

- **Thread safety** — `EllipsDInterface` reads the serial port in a daemon thread.  Callbacks are dispatched via a `frozenset` snapshot so adding a listener from inside a callback cannot deadlock.  Qt signals bridge the reader thread to the main (GUI) thread automatically via Qt's queued connection mechanism.
- **Body-frame convention** — X = forward, Y = right, Z = down.  The displayed speed is `LogEkfVelBody.velocity_x_ms` (signed; negative = reversing).
- **CRC scope** — Per the SBG ECom SDK (`sbgEComProtocol.c`): MSG_ID + CLASS + LEN_LO + LEN_HI + PAYLOAD.  The CLASS byte for all log messages is `0x00` (`SBG_ECOM_CLASS_LOG_ECOM_0`).
- **Auto-scaling gauge** — `compute_speed_range()` rounds the target to the nearest integer *in the display unit* to avoid unit-conversion artefacts, then applies the rule: target ≤ 5 → `[0, 2·t]`, target > 5 → `[0.5·t, 1.5·t]`.

## Installation (development)

```bat
:: Clone and create the virtual environment
git clone <repo>
cd Logger_speed
python -m venv .venv
.venv\Scripts\activate

:: Install the INS driver in editable mode + application dependencies
pip install -e ".[app]"
```

## Running from source

```bat
:: Default (INFO log level)
python run.py

:: Verbose output — useful for debugging the serial link
python run.py --verbose DEBUG
python run.py --verbose TRACE      :: logs every SBG frame received
```

## Building the executable

```powershell
# DEV build — increments the build counter
powershell -ExecutionPolicy Bypass -File build_exe.ps1

# DEV clean rebuild
powershell -ExecutionPolicy Bypass -File build_exe.ps1 -Clean

# Release builds — no DEV prefix, resets counter
powershell -ExecutionPolicy Bypass -File build_exe.ps1 -ReleaseRevision  # 1.0.x -> 1.0.x+1
powershell -ExecutionPolicy Bypass -File build_exe.ps1 -ReleaseMinor     # 1.x.0 -> 1.x+1.0
powershell -ExecutionPolicy Bypass -File build_exe.ps1 -ReleaseMajor     # x.0.0 -> x+1.0.0
```

The output is placed in `dist\`:

| Build type | Example exe name |
|---|---|
| DEV | `LoggerSpeed_DEV_1.0.0_build5.exe` |
| Release | `LoggerSpeed_1.0.1.exe` |

The exe is **self-contained** — copy it anywhere and run it.  No installation or admin rights required.  On first launch it creates a `log\` directory next to the exe.

## Diagnostic utilities

### `utils/raw_dump.py` — serial port diagnostic

Bypasses `sbg_ellipsd` entirely and reads raw bytes directly.  Use it to verify the serial link before running the full application:

```bat
python utils\raw_dump.py COM3
python utils\raw_dump.py COM3 921600 --hex     :: also prints hex dump of each frame
```

Interpretation guide:

| Observation | Likely cause |
|---|---|
| `bytes/s = 0` | Wrong port name or device not powered |
| `bytes/s > 0`, `sync = 0` | Wrong baud rate, or device not in binary mode |
| `sync > 0`, `CRC_OK = 0` | CRC scope mismatch (should not happen with current code) |
| `CRC_OK > 0` | Frames valid — issue is elsewhere in the application |

### `utils/test_listeners.py` — listener smoke test

Registers all 46 message listeners and prints every received message to the console:

```bat
python utils\test_listeners.py COM3
python utils\test_listeners.py COM3 921600 --verbose TRACE   :: all frames
python utils\test_listeners.py COM3 921600 --stats 10        :: stats every 10 s
```

## CSV log format

Logs are saved to `log\run_YYYYMMDD_HHMMSS.csv` (next to the exe when running as a bundle, or in the project root when running from source).

The file has **30 columns**:

| Column(s) | Content |
|---|---|
| `timestamp_iso` | Wall-clock time (ISO 8601, millisecond precision) |
| `elapsed_s` | Seconds since logging started |
| `speed_ms` | Forward speed in m/s — **signed** (negative = reversing) |
| `speed_kmh` | Same value converted to km/h |
| `latitude_deg`, `longitude_deg` | WGS84 coordinates (7 decimal places ≈ 1 cm) |
| `altitude_m` | Altitude above mean sea level |
| `solution_mode` | Text: `UNINITIALIZED` / `VERTICAL_GYRO` / `AHRS` / `NAV_VELOCITY` / `NAV_POSITION` |
| `attitude_valid` … `pos1_used` | 22 flag columns (0 or 1) decoded from the raw solution status |

## INS connection

| Parameter | Value |
|---|---|
| Default baud rate | **921 600** (matches ELLIPS-D factory default) |
| Frame format | 8N1, no flow control |
| Protocol | SBG ECom v5.6 binary, class `0x00` |
| Primary speed source | `LOG_EKF_VEL_BODY` (MSG `0x36`) — body-frame velocity |
| Position source | `LOG_EKF_NAV` (MSG `0x08`) — NED navigation solution |
